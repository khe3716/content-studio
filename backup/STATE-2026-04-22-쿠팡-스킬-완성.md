## 💡 다음 세션 우선순위

### 1순위: detail page generator의 config-driven화
`generate-detail-premium-v4.js`가 현재 두리안 텍스트·색상을 하드코딩. 새 상품마다 파일 복사·수정이 번거로움.

제안 구조:
```js
// generate-detail.js (리팩터)
const config = JSON.parse(fs.readFileSync(process.argv[2]));
// config에서 모든 섹션 텍스트 + 이미지 경로 + 컬러 팔레트 읽어서 렌더링
```

Config JSON 확장:
```json
{
  "design": {
    "palette": "amber" // amber | burgundy | orange | green 등 프리셋
  },
  "sections": {
    "hero": { ... },
    "empathy": { "headline_l1": "...", "headline_l2": "...", "body": "..." },
    "quality_points": [ { "tag": "01", "title": "...", "body": "..." }, ... ],
    "unboxing": { "headline": "이렇게 받으십니다", "body": [...] },
    "texture": { "headline": "크리미한 질감...", "tagline": "..." },
    "beginner_guide": [ { "emoji": "🥄", "title": "...", "body": "..." }, ... ],
    "owner_note": { "bullets": [...], "quote": "..." },
    "delivery_cards": [...],
    "notice": [...],
    "faq": [ { "q": "...", "a": "..." }, ... ],
    "closer": { "line1": "...", "line2": "..." }
  }
}
```

1. **Telegram 명령 추가 불필요** — 사진 업로드가 PC 필수라 Telegram 의미 없음.
2. **상품 이미지 업로드 API 없음** — GitHub Pages URL로 간접 업로드만 가능.
3. **카테고리별 필수 속성 다름** — 새 카테고리면 category-meta 조회 필요.
4. **수량 단위 고정** — "개", "박스", "세트"만 허용.
5. **정상가 최소 요구사항** — 판매가 == 정상가여도 OK이지만 할인 표시가 더 매력적.
