## AI Image Restyling (Gemini 2.5 Flash Image)

### 모델 정보

- **모델명**: `gemini-2.5-flash-image` (일명 Nano Banana)
- **엔드포인트**: `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash-image:generateContent`
- **API Key**: 기존 `GEMINI_API_KEY` 재사용 (블로그 생성에 이미 사용 중)
- **비용**: 무료 티어 내 (분당 ~10회, 일당 100회 한도)

### 제공 스타일 (restyle-image.js의 STYLE_PROMPTS)

| 스타일 | 설명 | 용도 |
|---|---|---|
| `minimal-nature` ⭐ | 린넨·원목·자연광·스칸디나비안 | 현재 사용 중 |
| `magazine-food` | 다크 슬레이트·브라스·드라마틱 | 대안 |
| `warm-home` | 우드보드·티컵·따뜻한 홈 | 대안 |
| `japanese` | 와비사비·돌·정갈함 | 대안 |
| `unboxing-editorial` | 포장 유지 + 브랜드 라벨 제거 | UNBOXING 섹션 전용 |
| `ultra-closeup` | 초근접 매크로·텍스처 | TEXTURE 섹션 전용 |

### 이미지 편집 프로세스

1. 원본 로드 → Sharp로 1024px 리사이즈 (API 페이로드 축소) → JPEG 90 인코딩
2. base64 인코딩 후 `inlineData`로 API 전송
3. 프롬프트: "KEEP ...", "CHANGE ..." 구조 (대상·변경 명확)
4. 응답의 `candidates[0].content.parts[].inlineData.data` 추출
5. base64 디코딩 → Sharp로 JPEG 92 저장

## Workflow (신규 상품 상세페이지 제작)

### 현재 프로세스 (수동, 상품별 1회)

1. 사용자가 `fruit-blog/detail-pages/source-photos/{상품명}/`에 원본 사진 드롭
2. `restyle-image.js`로 필요 사진 AI 연출컷 변환 (각 30초~1분)
3. `generate-detail-premium-v4.js` 수정 (상품명·컬러·카피)
4. `node generate-detail-premium-v4.js` 실행 (10~20초)
5. 출력 PNG를 네이버 스마트스토어 관리자에 수동 업로드

### 총 소요 시간

- 기초 스펙 파악 + 카피 작성: 15~30분
- 사진 준비 + AI 연출컷 변환: 10~15분
- 생성기 config 수정: 10~15분
- 총: **30~60분 / 상품**
