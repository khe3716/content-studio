## 6. Pipeline Flow (auto-publish.js)

```
[Trigger: cron OR manual]
   ↓
1. Read economy-blog/topics.yaml
2. Find first topic with status="ready" (or --day N arg)
3. Identify next topic for "다음 글 예고" line
   ↓
4. [김하나 persona + 3 sample drafts] → Gemini → 4000-5000 char HTML
5. Strip any <h2>Day N:...</h2> from top (anti-duplicate safeguard)
   ↓
6. [박팩트 persona] → Gemini → JSON validation report
7. If HIGH severity issues → revise via Gemini → save revised HTML
   ↓
8. Save to economy-blog/drafts/day-XX-slug.html
   ↓
9. Invoke publish-draft.js:
   a. Generate SVG (600x600, Korean text)
   b. Render to PNG at 1200x1200 (density 600)
   c. Composite MS Fluent 3D emoji at 310x310
   d. Downscale to 600x600 (Lanczos3)
   e. Save PNG
   f. base64-embed PNG as <img> at top of HTML
   g. POST to Blogger API (isDraft=true)
   ↓
10. Update topics.yaml: current → draft, next pending → ready
11. [CI only] git commit + push (uses GITHUB_TOKEN)
```

## 8. Known Limitations / Workarounds

### Blogger API Limitations
- **Cannot set custom permalink via API.** Must be done in Blogger web UI per-post.
- **Base64 images are auto-converted to CDN URLs** by Blogger. Regex-based thumbnail replacement must handle both `data:image/png;base64,...` AND `https://blogger.googleusercontent.com/...` patterns.
- **Status changes:** Updating a DRAFT may flip status to SCHEDULED. Use `force-thumbnail-refresh.js` which fetches fresh content from Blogger before updating.

### Node.js
- `get-refresh-token.js` is gitignored (intentional — one-time use).
- Node 20+ required for built-in `fetch`.

### Gemini API
- Sometimes outputs markdown code fences (\`\`\`html ... \`\`\`) — auto-stripped in post-processing.
- May prefix article with `<h2>📌 Day N: ...</h2>` — auto-stripped with regex.
- Free tier limits: Plenty of room for daily publishing; unlikely to hit caps.

### Microsoft Fluent Emojis
- 3D versions only available as PNG (no SVG). Source: https://github.com/microsoft/fluentui-emoji
- Native resolution: 256×256. Scaled to 155 via Lanczos3 (with 2x supersampling for anti-aliasing).

### Thumbnail

### Facts
- **Conservative wording always:** "상대적으로 유리할 수 있어요" instead of "훨씬 유리해요"
- Qualify generalizations: "일반적으로", "대체로"
- Date-sensitive claims: mark with "(2026년 4월 기준 예시)" or similar

### Forbidden Phrases
- "이거 사세요" / "무조건 오릅니다" / "역대급" / "완전"
- "지금 안 하면 망합니다" (fear-mongering)

## 11. Emergency Recovery Playbook

### If GitHub Actions fails
1. Check logs: `gh run list -R khe3716/content-studio` + `gh run view <ID> --log`
2. Common issues:
   - Gemini API rate limit → wait, retry
   - Blogger OAuth token expired → rerun `get-refresh-token.js`, update GitHub Secret
   - Malformed YAML → validate `economy-blog/topics.yaml`

### If refresh_token expires
```bash
cd "c:/antigravity/claude code"
node get-refresh-token.js
# Follow browser OAuth flow
# Updates .env automatically
# Then: gh secret set GOOGLE_REFRESH_TOKEN --body "$(grep GOOGLE_REFRESH_TOKEN .env | cut -d= -f2)" -R khe3716/content-studio
```

### If thumbnail doesn't update on Blogger
```bash
# Force refresh (handles CDN caching)
node force-thumbnail-refresh.js <POST_ID> economy-blog/thumbnails/day-XX.png
```

### If topics.yaml gets out of sync
- Manually edit status fields
- Each topic must have `day`, `cluster`, `title`, `thumb_title`, `subtitle[]`, `emoji`, `slug`, `labels[]`, `status`
- Valid statuses: `published`, `draft`, `ready`, `pending`
