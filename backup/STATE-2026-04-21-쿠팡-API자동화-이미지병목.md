## 🔑 중요 기술 발견

### 1. HMAC 서명 포맷
```
message = YYMMDDTHHmmssZ + METHOD + PATH + QUERY
Authorization: CEA algorithm=HmacSHA256, access-key={KEY}, signed-date={DT}, signature={SIG}
```
- datetime: UTC 기준, 형식 `YYMMDDTHHmmssZ`
- QUERY: 물음표 제외, 빈 문자열이면 그냥 빈 문자열
- Body는 서명에 포함 안 됨 (PATH + METHOD만 서명)

### 2. Akamai WAF (앞단 방어벽) 차단 패턴
- ❌ `raw.githubusercontent.com` URL 포함 body → 403 Access Denied
- ✅ `cdn.jsdelivr.net` → 통과 (Akamai 레벨)
- ✅ `upload.wikimedia.org` → 통과
- ✅ `httpbin.org` → 통과

**교훈**: GitHub raw URL을 body에 넣지 말 것. jsDelivr로 우회.

### 3. Coupang 내부 이미지 Fetcher (Akamai와 별개)
- Akamai를 통과해도 Coupang 서버가 **내부적으로 이미지 다운로드**하는 단계 별도 존재
- jsDelivr URL은 Akamai 통과했지만 Coupang 내부 fetcher가 다운 못 함 (`cdnPath: "-"`)
- 결과: 이미지 등록 실패 → 상품 승인반려
- **내일 해결해야 할 진짜 병목이 이 부분**

### 4. 쿠팡 공식 Open API 이미지 업로드 엔드포인트 **존재 안 함**
- 9개 후보 경로 전부 404:
  ```
  /v2/providers/marketplace_openapi/apis/api/v1/vendor/{vendorId}/image
  /v2/providers/marketplace_openapi/apis/api/v1/vendor/{vendorId}/image-upload
  /v2/providers/openapi/apis/api/v1/vendor/{vendorId}/image
  /v2/providers/openapi/apis/api/v1/vendor/{vendorId}/image-upload
  /v2/providers/seller_api/apis/api/v1/image
  /v2/providers/seller_api/apis/api/v1/image-upload
  /v2/providers/seller_api/apis/api/v1/marketplace/images
  /v2/providers/seller_api/apis/api/v1/vendor-image
  /v2/providers/seller_api/apis/api/v1/image/vendor-image
  ```
- 결론: Coupang 모델은 **"URL 주면 내가 다운로드"** 방식만 지원
- Wing UI는 내부 엔드포인트 사용 (세션 쿠키 기반, Open API로 접근 불가)

### 5. 필수 헤더 (POST 성공 확인된 것)
```
Authorization: (HMAC)
Content-Type: application/json;charset=UTF-8
X-EXTENDED-Timeout: 90000
```
- `X-Requested-By`, `User-Agent`, `Accept` 같은 헤더는 **영향 없음**

### 6. 옵션 속성 (attributes) 유효 단위
- `수량`: "1개", "1박스", "1세트" (개/박스/세트만 허용, "팩" ❌)
- `농산물 중량`: 자유 텍스트 문자열 ("깐 두리안 500g")
- MANDATORY 2개만 넣으면 됨 (나머지 OPTIONAL은 생략 가능)

### 7. `vendorUserId` 필수
- 상품 등록 시 `vendorUserId: "khe3714"` 필요
- 빠지면 "vendorUserId 값을 확인해 주세요" 에러
